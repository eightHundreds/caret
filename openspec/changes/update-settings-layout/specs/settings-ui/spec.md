## ADDED Requirements
### Requirement: 设置页分区导航
插件设置页 MUST 提供清晰的分区导航，将模型相关与聊天相关配置分离，避免一次性平铺全部字段。

#### Scenario: 默认展示模型与生成分区
- **WHEN** 用户打开插件设置页
- **THEN** 显示“模型与生成”分区内容，并隐藏“聊天与日志”等其他分区，用户可通过标签或按钮切换

#### Scenario: 切换到聊天配置
- **WHEN** 用户点击“聊天与日志”分区的切换控件
- **THEN** 聊天配置内容展示，模型与密钥分区收起且不会同时占据视图

### Requirement: 模型与 API 设置分组
模型/图像选择与 API 密钥 MUST 分组展示，优先显示当前供应商相关字段，减少不相关输入项的噪音。

#### Scenario: 默认模型分组
- **WHEN** 用户处于“模型与生成”分区
- **THEN** 顶部有“默认模型”分组，包含 LLM 提供商下拉、模型下拉和上下文窗口提示，切换提供商会更新模型列表与上下文提示

#### Scenario: 图像生成分组
- **WHEN** 用户处于“模型与生成”分区
- **THEN** 存在独立的“图像生成”分组，包含图像提供商和模型选择，并随着提供商切换刷新选项与选中值

#### Scenario: API 密钥分组按需显示
- **WHEN** 用户查看 API 密钥分组
- **THEN** 当前选中的 LLM 提供商的密钥输入优先展示，其他提供商的密钥输入折叠或置于次级分区，需要展开后才露出，避免长列表一次性平铺

#### Scenario: 本地模型提示
- **WHEN** 用户选择本地提供商（如 Ollama）
- **THEN** API 分组显示运行或启动提示文本块，指引如何拉取模型和启动服务

### Requirement: 聊天与日志设置分组
聊天历史与输入行为相关配置 MUST 集中在“聊天与日志”分区内，并按功能分组，避免与模型或密钥项混杂。

#### Scenario: 日志存储分组
- **WHEN** 用户位于“聊天与日志”分区
- **THEN** “日志存储”分组包含聊天保存路径、日期子目录开关、聊天重命名开关，输入校验与提示保持可用

#### Scenario: 输入与上下文分组
- **WHEN** 用户位于“聊天与日志”分区
- **THEN** “输入与上下文”分组包含发送快捷键选择和嵌套 [[]] 引用开关，切换时立即保存并不影响其他分组显示
